//метаданные
//другое название - owner - или просто schemes?
owner- таблица, в которой хранится имя пользователя. Собственник таблиц в которых хранятся данные для анализа.
scheme_id - идентификатор(первичный ключ)
scheme_name - имя пользователя

object_type - таблица, в которой хранится информация о типах объектов. Тип объекта это название таблицы в ГИС.
object_type_id - идентификатор(первичный ключ)
owner_id - ссылка на таблицу owner(внешний ключ)
object_name - имя типа объекта(имя таблицы)

attribute_type - таблица, в которой хранится информация о типах атрибутов. Тип атрибута-это название столбцов таблиц.
attribute_type_id - идентификатор(первичный ключ)
attribute_name - имя атрибута(имя столбца)
object_type_id - ссылка на таблицу object_type(внешний ключ)
data_type - храним тип переменной атрибута.

operation - таблица, в которой храним доступные операции. Бинарные операции выполняются между двумя типами объектов
operation_id - идентификатор(первичный ключ)
first_object_type_id - ссылка на таблицу object_type(внешний ключ). Первый тип объекта.
second_object_type_id - ссылка на таблицу object_type(внешний ключ). Второй тип объекта.
operation_name - название операции. Для удобства пользователю.
operation_description - описание операции.
operation_procedure - название функции в oracle, которая принимает два типа объекта и возвращает результат.

rule - таблица, в которой хранятся правила. Правило это множество операций.
rule_id - идентификатор правила(первичный ключ)
rule_name - название правила.
rule_description - описание правила.

rule_operation - таблица для связи правил и операций.
rule_id - ссылка на таблицу правил(внешний ключ)
operation_id - ссылка на таблицу операций(внешний ключ)
orderby - задает порядок выполнение операций в правиле.

КЭШ

session- таблица, в которой хранится информация по текущей сессии(анализа).
session_id - идентификатор сессии (первичный ключ)
creation_data - дата создания сессии.
session_description - описание сессии.

cache_object - таблица, в которой хранится информация об объектах, находящихся в данной области.
object_id - идентификатор(первичный ключ)
object_type_id - ссылка на таблицу object_type(внешний ключ)
object_value - значение первичного ключа таблицы object_type_id.

cache_attribute - таблица, в которой хранится значение атрибутов, выгружаемых с таблиц ГИС
cache_id - идентификатор кэша(первичный ключ)
session_id - ссылка на таблицу session(внешний ключ)
attribute_type_id - ссылка на таблицу attribute_type(внешний ключ)
object_id - ссылка на таблицу object(внешний ключ)
str_val - хранение для строковых значений.
number_val хранение для типов number
date_Val - хранение для дат.
