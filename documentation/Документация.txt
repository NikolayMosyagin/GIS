Определения:
под объектом будем подразумевать строку-кортеж из некоторой таблицы, в терминах нашей задачи являюще(надо дописать)

info_schemes-таблица, в которой хранится информация о схемах и представлениях, из которых далее будут извлекаться данные.
scheme_id - номер схемы
scheme_name - имя схемы

info_tables- таблица, в которой хранится информация о таблицах из соответствующих схем, из которых извлекаются данные
table_id - номер таблицы
table_name - имя таблицы
primary_column_name - имя столбца, который в данной таблице является первичным(примечание:можно будет убрать)
scheme_id - идентификационный номер схемы, в которой находится таблица

info_attributes - таблица, в которой хранится информация о столбцах из соответствующей таблицы, из которых мы будем извлекать данные.
attribute_id - номер таблицы
attribute_name - имя столбца
data_type - тип данных, которые хранятся в этом столбце
table_id - идентификационный номер таблицы, в которой находится данный столбец

info_objects - таблица, в которой хранится информация о кортежах, которые будут извлекаться из соответствующей таблицы.
object_id - номер объекта
table_id-идентификационный номер таблицы, к которой принадлежит данный столбец
table_object_id-идентификационный номер, который имеет данный кортеж в той таблице, из которой будут извлекаться данные

info_session-таблица, в которой хранятся данные о сессии.
session_id - номер сессии
creation_date-дата создания сессии

info_cache-таблица, в которую выгружаются данные, после выполнения соответствующей процедуры.
cache_id-идентификационный номер данного кортежа
session_id-номер сессии, когда эти данные были загружены
attribute_id-идентификационный номер столбца, из которого они были загружены
object_id-идентификационный номер объекта, к которому принадлежит данный атрибут
str_val, int_val, float_val, date_val-поля, предназначенные соответственно для строкового, целочисленного, вещественного типов и даты, в зависимости от того, 
какой тип данных имеет данный атрибут, в одном из этих полей хранится его значение, остальные остаются пустыми

operation-таблица, в которой хранится информация о доступных операциях. Операция выполняется между двумя некоторыми объектами. Для каждой операции 
существует процедура, реализующая данную операцию, на вход которой передается два указанных объекта.
id_operation-номер операции
id_firstObject-ид. номер первого объекта
id_secondObject-ид. номер второго объекта
operation_name-имя операции
procedure-имя процедуры, реализующей данную операцию
